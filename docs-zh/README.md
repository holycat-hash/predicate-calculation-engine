# 刁钻逻辑的切分方法集

**语言：** [English](../docs/README.md) | 中文

本目录收录**只靠 predicate + calculation + entity 切分**就能落地的刁钻逻辑实现方法。
规则与 PCE文档 §0 一致：四层封闭，任何需求折叠进四层，禁止引入第五种概念
（没有消息、没有事件总线、没有回调、没有全局函数）。

每篇的固定结构：**问题 → 为什么刁钻 → 切分 → 谓词代数 → 正确性论证（对照 D1/D2/D3 与快照读）→ 成本**。
谓词一律用文档 §7 的 DSL 风格书写。

## 通用手法速查

1. **物化为 entity**（§6.1）——join、空间查询、全局排序、个性化海量阈值，一律建索引实体：
   batch 订阅原始写流，增量维护，把视图写成自己的字段。
2. **事件实体化**——一个 cell 一帧只能折叠出一条写（§2 写折叠）；要在同帧发多条事件、
   或让多个接收者各取所需，就 spawn 一个实体当事件，出生（`_alive became(true)`）即广播。
3. **回执经 inst ref**——condition 不能引用「别人的行」，但自己持有的 ref 可以经
   `inst(ref, field)` 精确盯一个实例：请求方留 ref，等被请求方把结果写进自己字段。
4. **时间戳守卫**——「不早于」「冷却中」不要轮询：写事件时盖上 `Clock.frame` 快照戳，
   接收侧用 `cmp(new.frame, ≥, own.xxx_until)` 在谓词层挡掉。
5. **缺席靠租约**——「没发生」不可订阅（§3.3 否定限制）；改为正事件维护 `lease_until`，
   由唯一合法的轮询者（订阅 `Clock.frame`）或集中化的 Alarm 实体判到期。
6. **翻转设计信号**——某 cell 被海量个性化条件订阅（§4 诚实退化条款）= 该把中间量
   物化为 entity 的信号，不是优化索引的信号。
7. **op 同形合流**——scope 并（`|`）的各分支必须交付同形负载，单一 condition 才对
   所有分支同义；异形来源先由各自 calc 归一化成同形 op 字段再合流（§1.4 出路三的形状纪律）。
8. **静默拒绝 vs 回执拒绝**——谓词层守卫的拒绝免费但无回执；要反馈就拆互补守卫
   双谓词分流（一条进结算、一条进回执），或把拒绝下移进 calculation。
9. **own 流恰好一次**——D1 单写者+写折叠 ⇒ 每 cell 每帧至多一写 ⇒ own 流上边沿守卫
   即 exactly-once；扇入流的「恰好 N」必经 单写者计数 + batch + 多重集内裁决。
10. **calc 内全序重排**——D3 只是不承诺交付序；calculation 对收到的多重集按业务
    全序键排序再处理，是恢复「逐个语义」与确定性的合法手段（输出仍是多重集的函数）。
11. **已付轮询折叠**——实体本就每帧醒（动画 / 物理订阅 `Clock.frame`）时，同实体的
    窗口判定、到期检查、意图消费折叠进该驱动器的同一次运行是免费的：边沿机关退化为
    状态比对（恰好一次 = 单调 seq 一行不等式），重启 / 换挡天然原子；谓词边沿 + alarm
    版只留给没有每帧驱动器的实体（[14](14-combo-cancel-buffer.md)）。
12. **本地时间轴**——时间伸缩（顿帧 / 慢放 / 时停）下，「N 帧」语义一律挂到实体自己的
    `local_time` 单调字段：timekeeper 单写者整数分数累加推进，冻结 = 停轴 O(1)（顺延是
    推论，不枚举挂起戳），所有时间戳守卫（手法 4）只换轴不换形；跨实体判定用**被判定者**
    的轴（[22](22-time-dilation.md)）。
